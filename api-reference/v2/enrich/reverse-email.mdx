---
title: 'Reverse Email Lookup'
api: 'POST /v2/enrich'
description: 'Identify a person from their professional email address.'
---

## Overview

Reverse-resolves a professional email address into a person's identity:

1. **Parse** the local part with GPT to extract first name + last name + company domain
2. **Search Google** if the parse only yielded a first name (e.g. `j.doe@...`) to find the full name
3. **Locate the LinkedIn profile** via Google search using `name + company`
4. **Extract the full LinkedIn profile** for the matched URL

<Note>
Only **professional** email addresses are accepted. Personal providers (Gmail, Yahoo, Hotmail, Outlook, iCloud, ProtonMail, free.fr, orange.fr, etc.) are rejected with `INVALID_PARAMS`.

This endpoint consumes **1 credit** per request.
</Note>

## Request

<ParamField body="action" type="string" required>
  Must be `"reverse_email"`.
</ParamField>

<ParamField body="params" type="object" required>
  <Expandable title="params properties">
    <ParamField body="email" type="string" required>
      Professional email address to reverse-resolve.
    </ParamField>
  </Expandable>
</ParamField>

<RequestExample>
```json
{
  "action": "reverse_email",
  "params": {
    "email": "jane.doe@acme-corp.com"
  }
}
```
</RequestExample>

## Response

<ResponseField name="success" type="boolean">
  Indicates whether the request was successful.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="email_parsed" type="object">
      Output of the GPT parsing step.
      <Expandable title="email_parsed properties">
        <ResponseField name="original_email" type="string">The input email (echo).</ResponseField>
        <ResponseField name="first_name" type="string">First name extracted from the local part.</ResponseField>
        <ResponseField name="last_name" type="string">Last name extracted (may be empty if the local part contained only the first name).</ResponseField>
        <ResponseField name="company_domain" type="string">The company domain (everything after `@`).</ResponseField>
        <ResponseField name="parsing_confidence" type="string">GPT's confidence in the parse: `"high"`, `"medium"`, or `"low"`.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="person_found" type="object">
      The resolved person.
      - When a LinkedIn profile is matched: full profile object (same shape as [Get Profile Info](/api-reference/v2/profiles/get-profile)) — `full_name`, `first_name`, `last_name`, `headline`, `location`, `summary`, `industry`, `experience`, `education`, `skills`, `profile_picture_url`, `connection_count`, etc.
      - When no LinkedIn match is found: minimal fallback `{full_name, first_name, last_name}`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="metadata" type="object">
  <Expandable title="metadata properties">
    <ResponseField name="action" type="string">The action that was performed (`"reverse_email"`).</ResponseField>
    <ResponseField name="credits_consumed" type="integer">Number of credits consumed (1).</ResponseField>
    <ResponseField name="timestamp" type="string">ISO 8601 timestamp of the response.</ResponseField>
  </Expandable>
</ResponseField>
