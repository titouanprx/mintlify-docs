---
title: 'Get Post Comments'
api: 'POST /v2/content'
description: 'Retrieve comments on a specific post.'
---

## Overview

Returns the comments on a specific LinkedIn post. Pagination is **offset-based** — set `count` for the page size and `offset` to skip ahead.

<Note>This endpoint consumes **1 credit per 10 results** returned. LinkedIn returns up to **10 comments per internal request**, so larger `count` values trigger multiple upstream calls automatically.</Note>

## Request

<ParamField body="account_id" type="string" required>
  The unique identifier of the account to use.
</ParamField>

<ParamField body="action" type="string" required>
  Must be `"get_comments"`.
</ParamField>

<ParamField body="params" type="object" required>
  <Expandable title="params properties">
    <ParamField body="post_url" type="string" required>
      The URL of the post to retrieve comments from.
    </ParamField>
    <ParamField body="count" type="integer" default={10}>
      Number of comments to return. Defaults to `10`. **Capped at `1000`** per call.
    </ParamField>
    <ParamField body="offset" type="integer" default={0}>
      Zero-indexed offset into the comments list. Defaults to `0`. Use `pagination.next_offset` from a previous response to fetch the next page.
    </ParamField>
  </Expandable>
</ParamField>

<RequestExample>
```json First page
{
  "account_id": "your-account-id",
  "action": "get_comments",
  "params": {
    "post_url": "https://www.linkedin.com/feed/update/urn:li:activity:7654321098765432100",
    "count": 50,
    "offset": 0
  }
}
```

```json Next page
{
  "account_id": "your-account-id",
  "action": "get_comments",
  "params": {
    "post_url": "https://www.linkedin.com/feed/update/urn:li:activity:7654321098765432100",
    "count": 50,
    "offset": 50
  }
}
```
</RequestExample>

## Response

<ResponseField name="success" type="boolean">
  Indicates whether the request was successful.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="comments" type="array">
      List of comments.
      <Expandable title="comment properties">
        <ResponseField name="comment_urn" type="string">LinkedIn URN of the comment. Use this with the `answer_comment` action.</ResponseField>
        <ResponseField name="tracking_id" type="string">Tracking ID required to reply to the comment via `answer_comment`.</ResponseField>
        <ResponseField name="comment_text" type="string">Plain-text content of the comment.</ResponseField>
        <ResponseField name="created_time" type="integer">Unix epoch (milliseconds) when the comment was posted.</ResponseField>
        <ResponseField name="commenter" type="object">
          <Expandable title="commenter properties">
            <ResponseField name="name" type="string">Display name.</ResponseField>
            <ResponseField name="profile_urn" type="string">LinkedIn URN of the commenter (`urn:li:fsd_profile:...`).</ResponseField>
            <ResponseField name="linkedin_url" type="string">Public LinkedIn profile URL.</ResponseField>
            <ResponseField name="occupation" type="string">Headline / current role.</ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="total_results" type="integer">Number of comments returned in this response.</ResponseField>
    <ResponseField name="total_available_results" type="integer">Total number of comments on the post reported by LinkedIn.</ResponseField>
    <ResponseField name="pagination" type="object">
      <Expandable title="pagination properties">
        <ResponseField name="offset" type="integer">Offset of the first comment in this response.</ResponseField>
        <ResponseField name="count" type="integer">Number of comments returned (mirrors `total_results`).</ResponseField>
        <ResponseField name="next_offset" type="integer">Offset to send as `offset` in the next request. `null` when exhausted.</ResponseField>
        <ResponseField name="has_more" type="boolean">`true` if more comments are available.</ResponseField>
        <ResponseField name="results_per_page" type="integer">Maximum batch size used per upstream request (10).</ResponseField>
        <ResponseField name="pages_fetched" type="integer">Number of internal upstream batches consumed.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="metadata" type="object">
  <Expandable title="metadata properties">
    <ResponseField name="action" type="string">The action that was performed (`"get_comments"`).</ResponseField>
    <ResponseField name="account_id" type="string">The account ID used for the request.</ResponseField>
    <ResponseField name="credits_consumed" type="integer">Number of credits consumed (1 per 10 results).</ResponseField>
    <ResponseField name="timestamp" type="string">ISO 8601 timestamp of the response.</ResponseField>
  </Expandable>
</ResponseField>
