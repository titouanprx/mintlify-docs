---
title: 'Extract Profile Reactions'
api: 'POST /v1/data/signals/profile/reactions'
description: 'Extract reactions from a LinkedIn profile (posts that the person has liked/reacted to)'
---

Extract reactions from a LinkedIn profile, showing posts that the person has liked or reacted to.

<Note>
  **Credit Usage:** 1 credit per 10 results (or fraction thereof). For example:
  - 1-10 results = 1 credit
  - 11-20 results = 2 credits
  - 300 results = 30 credits
</Note>


### Header Parameters

<ParamField header="x-api-key" type="string" required>
  Your API key
</ParamField>

### Body Parameters

<ParamField body="profile_url" type="string" required>
  URL of the LinkedIn profile to extract reactions from
</ParamField>

<ParamField body="proxy_country" type="string" default="FR">
  Country code for proxy selection. Available: (US, UK, FR)
</ParamField>

<ParamField body="total_results" type="integer" default="10">
  Number of reactions to retrieve
</ParamField>

<ParamField body="start_page" type="integer">
  Starting page number for pagination
</ParamField>

<ParamField body="end_page" type="integer">
  Ending page number for pagination
</ParamField>

<ParamField body="cursor" type="string">
  Pagination cursor to continue a previous search
</ParamField>

### Notes

- The API extracts reactions that the profile owner has made on other posts and comments
- Results are paginated using both page-based and cursor-based pagination
- The `reaction_time` field provides relative time (e.g., "8h", "1d") rather than absolute timestamps
- Each reaction includes detailed information about the reactor, the original post, and the comment (if applicable)
- The `reaction_target` field indicates whether the reaction was on a "POST" or "COMMENT"
- The `reaction_id` field contains LinkedIn URN identifiers for unique reaction identification
- Profile pictures and headlines are included when available
- Rate limits apply to profile reaction extraction requests
- The pagination object provides both page information and a cursor for fetching the next page of results
