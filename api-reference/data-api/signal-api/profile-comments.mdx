---
title: 'Extract Profile Comments'
api: 'POST /v1/data/signals/profile/comments'
description: 'Extract comments from a LinkedIn profile (comments that the person has posted)'
---

Extract comments from a LinkedIn profile, showing comments that the person has posted on other posts.

<Note>
  **Credit Usage:** 1 credit per request
</Note>

### Header Parameters

<ParamField header="x-api-key" type="string" required>
  Your API key
</ParamField>

### Body Parameters

<ParamField body="profile_url" type="string" required>
  URL of the LinkedIn profile to extract comments from
</ParamField>

<ParamField body="proxy_country" type="string" default="FR">
  Country code for proxy selection. Available: (US, UK, FR)
</ParamField>

<ParamField body="total_results" type="integer" default="10">
  Number of comments to retrieve
</ParamField>

<ParamField body="start_page" type="integer">
  Starting page number for pagination
</ParamField>

<ParamField body="end_page" type="integer">
  Ending page number for pagination
</ParamField>

<ParamField body="cursor" type="string">
  Pagination cursor to continue a previous search
</ParamField>

### Response

<ResponseField name="status" type="string">
  Request status (success/error)
</ResponseField>

  <Expandable title="Properties">

          <Expandable title="Post Author Properties">
            <ResponseField name="name" type="string">

            <ResponseField name="profile_url" type="string">

            </ResponseField>

            <ResponseField name="title" type="string">

          </Expandable>
        </ResponseField>

        <ResponseField name="replies" type="array">

            <ResponseField name="reply_text" type="string">

            <ResponseField name="created_time" type="string">

                </ResponseField>

          Whether there are more results available

      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Notes

- The API extracts comments that the profile owner has posted on other posts
- Results are paginated using cursor-based pagination
- The `cursor` parameter can be used to continue from where a previous request left off
- Comments include replies when available
- Post content may be truncated for privacy reasons
- Rate limits apply to profile comment extraction requests
- The pagination object provides a cursor for fetching the next page of results
