---
title: 'Extract Post Comments'
api: 'POST /v1/data/signals/posts/comments'
description: 'Extract comments from a specific LinkedIn post'
---

Extract detailed comments from a LinkedIn post, including information about who commented and the comment content.

<Note>
  **Credit Usage:** 1 credit per request
</Note>

### Header Parameters

<ParamField header="x-api-key" type="string" required>
  Your API key
</ParamField>

### Body Parameters

<ParamField body="post_url" type="string" required>
  URL of the LinkedIn post to extract comments from
</ParamField>

<ParamField body="proxy_country" type="string" default="FR">
  Country code for proxy selection. Available: (US, UK, FR)
</ParamField>

<ParamField body="total_results" type="integer" default="10">
  Number of comments to retrieve (used when not in pagination mode)
</ParamField>

<ParamField body="use_pagination" type="boolean" default="false">
  Use pagination mode instead of total results mode
</ParamField>

<ParamField body="start_page" type="integer">
  Starting page number for pagination mode (required if use_pagination is true)
</ParamField>

<ParamField body="end_page" type="integer">
  Ending page number for pagination mode (optional if use_pagination is true)
</ParamField>

### Notes

- The API supports LinkedIn post URLs in various formats
- Results are paginated with configurable page ranges when using pagination mode
- Comments include replies when available
- The profile picture URL may be null if not available
- Connection degree shows your network relationship with the comment author
- **Pagination Mode vs. Total Results Mode**:
  - **Total Results Mode**: Use the `total_results` parameter to specify how many comments you want (default)
  - **Pagination Mode**: Set `use_pagination` to true and use `start_page` and `end_page` parameters
- Rate limits apply to comment extraction requests
- The `total_available` field shows the total number of comments available on the post
