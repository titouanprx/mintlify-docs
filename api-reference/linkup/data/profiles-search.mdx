## Description

This endpoint allows you to search LinkedIn profiles using different search criteria.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Body Parameters

<ParamField body="keyword" type="string | array">
  Main search keyword(s) (name, position, skills, etc.)
  
  - Single string: Performs advanced search with filters
  - Array of strings: Automatically switches to batch mode for parallel searching
</ParamField>

<ParamField body="job_title" type="string | array">
  Job title(s) to search for
  
  Can be a single string or an array of job titles
</ParamField>

<ParamField body="industry" type="string | array">
  Industry sector(s)
  
  Can be a single string or an array of industries (e.g., ["technology", "finance", "healthcare"])
</ParamField>

<ParamField body="school" type="string | array">
  School(s) or university(ies)
  
  Can be a single string or an array of educational institutions
</ParamField>

<ParamField body="location" type="string | array">
  Location(s) (city, region, country)
  
  Can be a single string or an array of locations (e.g., ["Paris", "Lyon", "Marseille"])
</ParamField>

<ParamField body="current_company" type="string | array">
  Current company(ies)
  
  Can be a single string or an array of company names
</ParamField>

<ParamField body="num_results" type="integer" default="10">
  Number of results to return
  
  - Minimum: 1
  - Maximum: 50,000
  - Default: 10
</ParamField>

<ParamField body="total_results" type="boolean" default="false">
  Include total count of available results
  
  - `true`: Return total count of all matching results in response
  - `false`: Don't return total count (default)
</ParamField>



<Note>
At least one search criteria must be provided among: `keyword`, `job_title`, `industry`, `school`, `location`, or `current_company`.
</Note>

## Response

<ResponseField name="status" type="string">
  Response status ("success" or "error")
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Response Data">
    <ResponseField name="results_returned" type="integer">
      Number of results returned in this response
    </ResponseField>
    
    <ResponseField name="total_results" type="integer">
      Total count of all matching results (only included if total_results parameter is true)
    </ResponseField>
    
    <ResponseField name="profiles" type="array">
      List of found profiles
      <Expandable title="Profile Structure">
        <ResponseField name="url" type="string">
          LinkedIn profile URL
        </ResponseField>
        <ResponseField name="title" type="string">
          Profile title/name from LinkedIn
        </ResponseField>
        <ResponseField name="description" type="string">
          Profile description/summary
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Error message (if status = "error")
</ResponseField>

## Request Example

```python
import requests

url = "https://api.linkupapi.com/v1/data/search/profiles"

payload = {
    "keyword": "developer",
    "job_title": "Software Engineer",
    "industry": "Technology",
    "location": "Paris",
    "current_company": "Microsoft",
    "num_results": 15,
    "total_results": True
}
headers = {
    "x-api-key": "your-api-key-here",
    "Content-Type": "application/json"
}

response = requests.post(url, json=payload, headers=headers)
print(response.json())
```

```json
{
  "keyword": "developer",
  "job_title": "Software Engineer",
  "industry": "Technology",
  "location": "Paris",
  "current_company": "Microsoft",
  "num_results": 15,
  "total_results": true
}
```

## Error Codes

<ResponseExample>
```json
{
  "status": "error",
  "message": "At least one search criteria must be provided"
}
```
</ResponseExample>

- **400**: Invalid parameters or no search criteria provided
- **401**: Invalid API key or insufficient credits
- **500**: Server error during request processing

```json
}
```

## Important Notes

- Each successful search consumes 1 credit (single keyword) or 1 credit per keyword (multiple keywords)
- Credits are only deducted on successful searches
- The endpoint uses an automatic retry system for proxy errors
- **Single keyword**: Advanced search with optional filters for precise results
- **Multiple keywords**: Automatically activates batch mode for parallel searching
- Combining multiple criteria improves result quality
- All search parameters support both single strings and arrays for flexible searching
- When using multiple keywords, batch mode is automatically activated for optimal performance
- Set `total_results: true` to get the total count of all matching results in the response
