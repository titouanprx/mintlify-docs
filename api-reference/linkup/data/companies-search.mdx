---
title: "Search Companies"
api: "POST /data/search/companies"
description: "Search LinkedIn company pages with advanced filters"
---

## Description

This endpoint allows you to search LinkedIn company pages using various advanced filtering criteria. It uses Google Search to identify LinkedIn company profiles matching the specified criteria.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Body Parameters

<ParamField body="keyword" type="string" required>
  Keyword to search for (company name, industry, etc.)
  
  Can also be provided as `company_name` for backward compatibility
</ParamField>

<ParamField body="num_results" type="integer" default="10">
  Number of results to return
  
  - Minimum: 1
  - Maximum: 50,000
  - Default: 10
</ParamField>

<ParamField body="country" type="string" default="FR">
  Country code for proxy (e.g., "FR", "US", "GB")
</ParamField>

<ParamField body="filters" type="object">
  Advanced filters to refine the search
  
  <Expandable title="Filter Structure">
    <ParamField body="filters.industry" type="object">
      Industry sector
      - `names`: List of industries to include
      - `exclude`: List of industries to exclude
    </ParamField>
    
    <ParamField body="filters.location" type="object">
      Company location
      - `city`: City
      - `region`: Region
      - `country`: Country
    </ParamField>
    
    <ParamField body="filters.company_size" type="object">
      Company size
      - `ranges`: Company size ranges
      - `employee_count`: Specific employee count
    </ParamField>
    
    <ParamField body="filters.funding" type="object">
      Funding information
      - `funding_stage`: Funding stage (seed, series_a, series_b, series_c)
    </ParamField>
    
    <ParamField body="filters.founding_date" type="object">
      Founding date
      - `year`: Founding year
      - `year_range`: Year range
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status ("success" or "error")
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Response Data">
    <ResponseField name="keyword_searched" type="string">
      The keyword used for the search
    </ResponseField>
    
    <ResponseField name="filters_applied" type="object">
      The filters applied to the search
    </ResponseField>
    
    <ResponseField name="query_used" type="string">
      The generated Google Search query
    </ResponseField>
    
    <ResponseField name="total_available" type="integer">
      Total number of available results
    </ResponseField>
    
    <ResponseField name="results_returned" type="integer">
      Number of results returned in this response
    </ResponseField>
    
    <ResponseField name="companies" type="array">
      List of found companies
      <Expandable title="Company Structure">
        <ResponseField name="name" type="string">
          Company name
        </ResponseField>
        <ResponseField name="linkedin_url" type="string">
          Company LinkedIn profile URL
        </ResponseField>
        <ResponseField name="description" type="string">
          Company description
        </ResponseField>
        <ResponseField name="industry" type="string">
          Industry sector
        </ResponseField>
        <ResponseField name="location" type="string">
          Company location
        </ResponseField>
        <ResponseField name="employee_count" type="string">
          Number of employees
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Error message (if status = "error")
</ResponseField>

## Request Example

```json
{
  "keyword": "technology",
  "num_results": 20,
  "country": "FR",
  "filters": {
    "industry": {
      "names": ["technology", "software"],
      "exclude": ["consulting"]
    },
    "location": {
      "city": "Paris",
      "country": "France"
    },
    "company_size": {
      "ranges": ["11-50", "51-200"]
    },
    "funding": {
      "funding_stage": "series_a"
    }
  }
}
```

## Response Example

<ResponseExample>
```json
{
  "status": "success",
  "data": 
    {
    "results_returned": 20,
    "companies": [

        "industry": "Technology",
        "location": "Paris, France",
        "employee_count": "11-50"

    ]

```
</ResponseExample>

## Error Codes

<ResponseExample>
```json
{
  "status": "error",
  "message": "keyword (or company_name) is required"
}
```
</ResponseExample>

- **400**: Invalid or missing parameters
- **401**: Invalid API key or insufficient credits
- **500**: Server error during request processing

## Important Notes

- Each successful search consumes 1 credit
- Credits are only deducted on successful searches
- The endpoint uses an automatic retry system for proxy errors
- The `company_name` parameter is supported for backward compatibility with `keyword`
