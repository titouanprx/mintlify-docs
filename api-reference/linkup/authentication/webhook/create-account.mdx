---
title: 'Create Webhook Account'
api: 'POST /webhooks/accounts'
description: 'Register a LinkedIn account for webhook monitoring'
---

Register a new LinkedIn account to receive real-time webhook notifications for messages.

<Note>
  After creating an account, monitoring must be started separately using the start monitoring endpoint.
</Note>

<Warning>
  **Credit Usage:** Real-time monitoring consumes **10 credits per day** (1 credit every 144 minutes) per active account. This is a resource-intensive feature but extremely powerful for real-time LinkedIn messaging. We are continuously working to optimize costs.
</Warning>

### Header Parameters

<ParamField header="x-api-key" type="string" required>
  Your API key
</ParamField>

### Body Parameters

<ParamField body="login_token" type="string" required>
  LinkedIn authentication cookie (li_at) obtained from the login process
</ParamField>

<ParamField body="webhook_url" type="string" required>
  URL where webhook notifications will be sent
</ParamField>

<ParamField body="country" type="string" default="FR">
  Country code for proxy selection. Available: (US, UK, FR)
</ParamField>

<ParamField body="account_name" type="string">
  Optional name to identify this account
</ParamField>

### Response

<ResponseField name="id" type="string">
  Unique identifier for the created account
</ResponseField>

<ResponseField name="login_token" type="string">
  The LinkedIn authentication token
</ResponseField>

<ResponseField name="country" type="string">
  Country code used for this account
</ResponseField>

<ResponseField name="webhook_url" type="string">
  URL where webhooks will be sent
</ResponseField>

<ResponseField name="account_name" type="string">
  Account name if provided
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether monitoring is currently active (false by default)
</ResponseField>

<ResponseField name="created_at" type="string">
  Account creation timestamp
</ResponseField>

<ResponseField name="updated_at" type="string">
  Last update timestamp
</ResponseField>

### Notes
- The account will be created with `is_active: false` by default
- Use the start monitoring endpoint to begin receiving webhooks
- Credit consumption only begins when monitoring is started (not when account is created)
- Each active account consumes credits independently
