This tutorial walks you through building a complete LinkedIn outreach campaign — from connecting an account to tracking replies. Whether you're building an AI agent, a SaaS product, or an internal tool, this flow is the foundation of any LinkedIn automation.

## Campaign Flow Overview

Here's the full lifecycle of an automated LinkedIn campaign:

<Steps>
  <Step title="Connect Account">
    Authenticate a LinkedIn account via the Login endpoint.
  </Step>
  <Step title="Setup Webhooks">
    Configure `new_relation` and `new_message` events to receive real-time notifications.
  </Step>
  <Step title="Search Leads">
    Find your target audience using profile search.
  </Step>
  <Step title="Send Invitations">
    Send personalized connection requests with proper pacing.
  </Step>
  <Step title="Detect Accepted Invitations">
    Your webhook receives a `new_relation` event when someone accepts.
  </Step>
  <Step title="Send Message">
    Message the new connection after a natural delay.
  </Step>
  <Step title="Track Replies">
    Your webhook receives a `new_message` event when they respond.
  </Step>
  <Step title="Follow-Up Sequence">
    Automatically follow up if no reply after a few days.
  </Step>
</Steps>

## Step 1 — Connect a LinkedIn Account

First, authenticate a LinkedIn account via the Login endpoint.

```bash
curl -X POST https://api.linkupapi.com/v2/login \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "platform": "linkedin",
    "login_token": "AQEDAxxxxxxx",
    "country": "FR"
  }'
```

```json
{
  "success": true,
  "data": {
    "account_id": "acc_abc123",
    "status": "active"
  }
}
```

<Note>
  If the account requires 2FA, you'll receive a `CHECKPOINT_REQUIRED` response. Complete it via the [Checkpoint](/api-reference/v2/accounts/checkpoint) endpoint before continuing.
</Note>

Save the `account_id` — you'll use it in every subsequent request.

---

## Step 2 — Setup Webhooks

Before sending any invitations, set up webhooks to be notified in real-time when events happen (invitation accepted, new message, etc.).

```bash
curl -X POST https://api.linkupapi.com/v2/webhooks \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": "acc_abc123",
    "url": "https://your-app.com/webhooks/linkup",
    "events": ["new_relation", "new_message"]
  }'
```

Then start monitoring:

```bash
curl -X POST https://api.linkupapi.com/v2/webhooks/{webhook_id}/start \
  -H "x-api-key: YOUR_API_KEY"
```

**Key events for campaigns:**
| Event | When it fires |
|---|---|
| `new_relation` | A connection request is accepted |
| `new_message` | A new message is received in any conversation |

## Step 3 — Search for Leads

Use the search endpoint to find your target audience.

```bash
curl -X POST https://api.linkupapi.com/v2/profiles \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": "acc_abc123",
    "action": "search_people",
    "params": {
      "keywords": "CTO SaaS",
      "location": "France",
      "limit": 25
    }
  }'
```

The response returns a list of profiles with their `profile_url` — store these as your lead list.

<Tip>
  LinkedIn caps search results at **1,000 profiles per query**. To get more results, use different filter combinations (location, industry, company size) and run multiple searches.
</Tip>

---

## Step 4 — Send Connection Requests

Loop through your lead list and send personalized invitations.

```bash
curl -X POST https://api.linkupapi.com/v2/network \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": "acc_abc123",
    "action": "invite",
    "params": {
      "profile_url": "https://www.linkedin.com/in/johndoe",
      "message": "Hi John, I saw your work at Acme Corp — would love to connect."
    }
  }'
```

<Warning>
  **Respect LinkedIn limits.** For paid accounts: 80-100 invitations/day, ~200/week. For free accounts: much lower. See [Platform Best Practices](/api-reference/v2/best-practices) for details.

  Space out invitations with **random delays** (30s–2min between each) to mimic human behavior.
</Warning>

**Example pacing logic (Python):**

```python
import time
import random

for lead in leads:
    send_invitation(lead["profile_url"], personalized_message(lead))
    delay = random.uniform(30, 120)  # 30s to 2min
    time.sleep(delay)
```

## Step 5 — Detect Accepted Invitations

When someone accepts your invitation, your webhook receives a `new_relation` event:

```json
{
  "event": "new_relation",
  "account_id": "acc_abc123",
  "data": {
    "profile_url": "https://www.linkedin.com/in/johndoe",
    "name": "John Doe",
    "headline": "CTO at Acme Corp"
  },
  "timestamp": "2026-03-26T14:30:00Z"
}
```

**In your webhook handler**, mark this lead as "connected" and trigger the next step:

```python
@app.post("/webhooks/linkup")
async def handle_webhook(payload: dict):
    if payload["event"] == "new_relation":
        profile_url = payload["data"]["profile_url"]

        # Mark lead as connected in your database
        await db.leads.update(
            {"profile_url": profile_url},
            {"status": "connected", "connected_at": payload["timestamp"]}
        )

        # Schedule the first message (with a delay)
        await schedule_message(profile_url, delay_hours=random.uniform(1, 4))

    return {"status": "ok"}
```

<Tip>
  Don't send a message immediately after acceptance — wait **1-4 hours** to look natural.
</Tip>

---

## Step 6 — Send the First Message

Once a lead has accepted and the delay has passed, send your message:

```bash
curl -X POST https://api.linkupapi.com/v2/messages \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": "acc_abc123",
    "action": "send",
    "params": {
      "profile_url": "https://www.linkedin.com/in/johndoe",
      "message": "Thanks for connecting John! I noticed Acme Corp is scaling its engineering team. We help SaaS companies like yours with..."
    }
  }'
```

## Step 7 — Track Replies

When the lead responds, your webhook receives a `new_message` event:

```json
{
  "event": "new_message",
  "account_id": "acc_abc123",
  "data": {
    "conversation_id": "conv_xyz789",
    "sender": {
      "profile_url": "https://www.linkedin.com/in/johndoe",
      "name": "John Doe"
    },
    "message": "Hey, thanks for reaching out! Tell me more."
  },
  "timestamp": "2026-03-26T18:45:00Z"
}
```

**In your webhook handler**, you can:
- Mark the lead as "replied" in your CRM
- Forward the reply to your sales team
- Use AI to generate a contextual response

```python
@app.post("/webhooks/linkup")
async def handle_webhook(payload: dict):
    if payload["event"] == "new_message":
        sender = payload["data"]["sender"]["profile_url"]
        message = payload["data"]["message"]

        # Update lead status
        await db.leads.update(
            {"profile_url": sender},
            {"status": "replied", "last_reply": message}
        )

        # Option: Auto-respond with AI
        # response = await generate_ai_reply(message, lead_context)
        # await send_message(sender, response)

    return {"status": "ok"}
```

## Step 8 — Follow-Up Sequence

If a lead doesn't reply, schedule follow-ups:

```python
async def follow_up_sequence(profile_url: str):
    follow_ups = [
        {"delay_days": 3, "message": "Hi again! Just wanted to follow up on my last message..."},
        {"delay_days": 7, "message": "Last check-in — happy to share more details if you're interested."},
    ]

    for follow_up in follow_ups:
        await asyncio.sleep(follow_up["delay_days"] * 86400)

        # Check if lead has replied in the meantime
        lead = await db.leads.find_one({"profile_url": profile_url})
        if lead["status"] == "replied":
            return  # Stop sequence

        await send_message(profile_url, follow_up["message"])
```

<Warning>
  Limit follow-ups to **2-3 max**. More than that is spammy and can get your account flagged.
</Warning>

## Full Architecture

Here's how it all fits together in a typical SaaS or AI agent setup:

<CardGroup cols={2}>
  <Card title="Lead Database" icon="database">
    Stores each lead's `profile_url`, current status, and timestamps. No need to duplicate profile data — use the `get_profile` action to fetch it on demand.
  </Card>
  <Card title="Scheduler / Queue" icon="clock">
    Manages timing and pacing of all actions with random delays between each request.
  </Card>
  <Card title="Webhook Handler" icon="webhook">
    Receives real-time events from LinkupAPI and triggers the next step in your campaign flow.
  </Card>
  <Card title="AI Layer (optional)" icon="sparkles">
    Call `get_profile` to get full context on a lead, then use AI to generate a personalized message based on their headline, company, experience, etc.
  </Card>
</CardGroup>

### What to Store Per Lead

You only need to track the campaign state — profile details can be fetched live via the [Get Profile](/api-reference/v2/profiles/get-profile) action whenever you need context (e.g., before generating a personalized message).

| Field | Description | Example |
|---|---|---|
| `profile_url` | LinkedIn profile URL (unique identifier) | `https://linkedin.com/in/johndoe` |
| `account_id` | Which LinkedIn account owns this lead | `acc_abc123` |
| `campaign_id` | Which campaign this lead belongs to | `camp_001` |
| `status` | Current step in your flow | `searched`, `invited`, `connected`, `messaged`, `replied` |
| `invited_at` | When the invitation was sent | `2026-03-26T10:00:00Z` |
| `connected_at` | When the invitation was accepted (from webhook) | `2026-03-26T14:30:00Z` |
| `messaged_at` | When the first message was sent | `2026-03-26T18:00:00Z` |
| `replied_at` | When the lead replied (from webhook) | `2026-03-27T09:15:00Z` |
| `follow_up_count` | Number of follow-ups sent | `1` |

<Tip>
  When it's time to send a message, call `get_profile` to get fresh data (headline, company, experience) and use that as context to personalize your message — either with templates or AI.
</Tip>
